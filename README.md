# XPILE — Multi-Language Transpiler

> Transpile Python, C, C++, and Java into all four languages simultaneously — with a live IR viewer, token stream, and side-by-side diff view.

---

## 🚀 How to Run

### Prerequisites
- **GCC** (to compile the transpiler engine)
- **Python 3** (to run the local server)


### Manual steps to launch using terminal

```bash
# Step 1 — compile the transpiler engine
gcc transpiler.c -o transpiler 

# Step 2 — start the server
python3 server.py
```

### Open in your browser

| Page | URL |
|---|---|
| Landing page | http://localhost:8080/ |
| Transpiler app | http://localhost:8080/app.html |

## How It Works

The transpiler processes source code through a 5-stage compilation pipeline:

```
Source Code → [1] Lexer → [2] Parser → [3] IR Build → [4] Code Gen → [5] Output
```

1. **Lexer** — tokenises the source into a flat `Token[]` stream
2. **Parser** — builds a language-agnostic `IRNode[]` (Intermediate Representation)
3. **IR Build** — resolves types, scopes, and operators into a universal IR
4. **Code Gen** — four independent generators emit Python, C, C++, and Java
5. **Output** — a single JSON object is returned to the frontend

The C binary communicates via **stdin/stdout**:
- **Input:** `<lang>\n<source code>`
- **Output:** `{ tokens, ir, python, c, cpp, java, srclang, errors }`

The Python server (`server.py`) acts as a thin bridge — it receives `POST /transpile` requests from the browser, pipes them to the binary, and returns the JSON response.
